# 智慧物业管理系统

本系统是一套基于 Django (MVC/MVT) 架构开发的前后端不分离物业管理系统，界面使用 Bootstrap 5 结合自定义 CSS 开发，追求设计现代、响应式良好并兼容移动端及 PC 端的访问。通过了完整的 100% 容器化规范，实现极速上手配置。

## 🛠 技术栈
- 框架: Django 4.2
- 数据库: MySQL 8.0
- 前端UI: Bootstrap 5 + 纯原生的现代化响应式界面
- 容器化部署: Docker / Docker-Compose 完整封装

## How to Run
1. 确保系统安装有 Docker 及 Docker Desktop 已启动。
2. 在项目根目录执行：
   ```bash
   docker compose up -d --build
   ```
3. 首次启动时，数据库需要大约 10-20 秒来进行初始化，后端容器会自动延时等待、执行数据库迁移命令，并自动填充测试数据。
4. 等待容器启动完成...

## Services
- Web 访问地址 (前端与后端一体): [http://localhost:18000](http://localhost:18000)
- Database: localhost:13306 

## Verification
通过以下简单步骤，即可验证项目的核心功能：
1. **测试物业人员后台权限**：使用管理员账号 `admin` 或 `staff` 登录 http://localhost:18000 ，检查能否看到系统级别的大盘数据并访问左侧列表的“楼盘管理”。
2. **测试楼盘与房开管理功能**：点击“新增楼盘”，输入任意数据信息，点击提交并在楼盘列表中确认数据被正确创建成功，随后验证表单的错误校验（如留白提交时的浏览器原生校验以及前端反馈）。
3. **测试业主登录与多角色**：退出当前账户，使用 `owner1` 或 `owner2` 账号登录页面。您将观察到侧边栏权限菜单已被限制隐藏，且“系统大盘”被转化为业主的“我的房产”。
4. **测试提交报修流转与数据穿透**：业主在后台递交一条新的“测试报修”。退出后重新用 `staff` 验证，能观察到相关申请跳入了管理人员的“待处理数据流”中。
5. **测试物业费用账单**：使用 `staff` 后台在“费用中心”点击一键新增账单，指定发给 `owner1`。随后 `owner1` 登录就可以在首页“近期缴费提醒”中立即观察到此账单出现，并且在费用清单列表中体现它为 **未支付(unpaid)** 状态。

## 🌟 系统核心功能模块

### 1. 物业基础资料管理模块
- **楼盘与房态管理**: 包含楼盘(Estate)、楼栋(Building)、楼层(Floor)与单元(Unit)四级联动管理，清晰规划社区房产。
- **业主信息管理**: 建立用户配置与档案，将房号与业主信息（如 `owner1` 对接 101 室）相挂钩。

### 2. 报修管理模块
- **在线报修提交**: 支持业主快速递交水电维修、门窗损坏报单，提供完整描述和类型记录。
- **物业处理调度**: 后台系统向管理人员展示报修中心大盘，能直接点击处理调度单，配置跟进流。
- **意见反馈机制**: 完工后带有后续反馈信息归档，提供高透明度服务。

### 3. 费用管理模块
- **多维度分期账单**: 全面容纳物业费、水电费不同科目，并配置不同的付款截止期。
- **自动化追踪展示**: 业主可以在自己的后台首页直接查看催缴情况并记录状态。

## 🛡️ 工程质量和交付标准体现
由于遵守 Prompt2Repo 开发规范：
- 本项目严禁“非真实”的 `sqlite` 本地依赖，默认开启线上级配置的 MySQL 8.0 。
- 100% 遵守 `utf8mb4` 以及北京时间。
- Docker 服务均实现了网络互通(Service Name: `db`)，绝没依赖 `localhost` 写死数据库配置。
